Première utilisation

Bienvenue dans l'administration. Avant d'accéder aux fonctions d'édition, définissez un mot de passe local. Il vous sera demandé à chaque visite, en plus du jeton GitHub.
Pourquoi un mot de passe ici ? Ce mot de passe n'est pas un mécanisme de sécurité fort (il est stocké en clair dans votre navigateur). Il sert à protéger l'accès contre un regard par-dessus l'épaule ou un partage informel de l'URL. La vraie sécurité de la publication reste assurée par votre jeton GitHub.

Authentification

Pour publier des modifications, un jeton GitHub personnel est requis. Il sera stocké uniquement dans votre navigateur (jamais transmis à un serveur tiers).
Comment obtenir un jeton GitHub ?
  1. Allez sur github.com/settings/tokens (jetons à granularité fine)
  2. Cliquez sur Generate new token → Fine-grained personal access token
  3. Nom : CYIM-Simulateur-Admin, expiration : 1 an (ou personnalisé)
  4. Resource owner : votre compte personnel
  5. Permissions : sous Account permissions, activez Gists → Read and write
  6. Générez le jeton et copiez-le immédiatement (il n'est affiché qu'une fois)
⚠ Sécurité : ce jeton donne accès à vos Gists GitHub. Ne le partagez avec personne. Si vous l'utilisez sur un poste partagé, déconnectez-vous en fin de session (bouton « Déconnexion » en haut à droite).
